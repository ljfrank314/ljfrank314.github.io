PRIMROSE - Lunar Excavation Rover

ANSYS • ONSHAPE • Team Leadership • Rapid Prototyping

My Role: Chassis Team Lead

Joined the Planetary Surface Technology Development Lab (PSTDL) after Level 1 design, leading the 3-person subteam through fabrication, testing, and complete redesign of a 375 kg lunar excavation rover's chassis. Responsible for integration of all structural systems into rover as well as a novel drivetrain design.

Level 1 design.

Level 2 design. Changes limited to enabling manufacturability.

Level 3 design. Complete overhaul of rover, including drivetrain.

Design Philosophy: Speed Through Simplicity

Developed a manufacturing hierarchy to maximize build speed with limited resources:

  • 3D printed if possible (brackets, spacers)
  • Bent sheet metal + rivets (no welding required)
  • Bent sheet metal + welded joints
  • Square tubing + welding
  • In-house machining utilized only when unavoidable

This approach let us build fast and iterate faster - critical when discoveries during integration demanded immediate redesigns. Even the most expensive option, making billet parts, was done in house by team members to save cost.

Level 2: Bring the rover to life

Inherited a conceptual design that needed manufacturability fixes. Built the steel frame without time for FEA - pure experience-based design. Key contributions:

  • Resolved geometric interferences in CAD before cutting metal
  • Designed serviceable subassemblies (learned from automotive work)
  • Modified existing suspension hardware for 15-day endurance test
  • Result: 30,063m traversed, 2987.2 kg excavated and dumped, albeit with downtime for maintenance

Level 2 rover during endurance test showing original configuration

Level 3: Huntsville or Bust

Worked with chief engineer and team to set new requirements: wider wheelbase for stability, larger wheels to prevent sinking in sand or regolith, 30°/30° steering capability. Key contributions:

Structural Redesign:

  • Converted internal frame from steel to aluminum (weight savings + new electrical system)
  • Added frame brackets to accommodate larger wheels and wider wheelbase
  • ANSYS FEA with aggressive FoS (verified against hand calcs)
  • Maintained top-down CAD architecture in Onshape for team coordination

ANSYS quasi-static analysis with frame constrained by free sliding wheels

Suspension Overhaul:

  • Adapted existing kinematic layout geometry to new wheel envelope
  • redesigned A-arm mounts and most suspension components
  • Modified frame to support extended wheelbase and larger wheels
  • Implemented 30°/30° steering angle

New suspension assembly showing wider track width and improved mounting

Integration Under Fire

Fast design meant some components were modeled just days before fabrication. This required:

  • Constant communication with excavation/electrical teams
  • On-the-fly modifications when reality didn't match CAD
  • Balancing "perfect" vs "good enough" to meet deadlines

The payoff: 85% excavation rate improvement enabled by a chassis that could handle the increased loads, and 100% drivetrain reliability.

Final Results & Reflection

The rover secured 3rd place overall at NASA's Break the Ice Lunar Challenge, validating our scrappy approach against teams with significant industry backing. More importantly, this project transformed my engineering practice. The PSTDL gave me the freedom to fail fast and get hands on. Working alongside a team that trusted an undergrad to work as an engineer redesigning and designing critical systems taught me that good engineering isn't about perfection - it's about making the right compromises under pressure.

IoT Sensor Network

C++ • MQTT • PostgreSQL • Docker

Overview

Designed and deployed a scalable IoT sensor network for environmental monitoring across multiple sites. The system collects temperature, humidity, air quality, and noise level data from 50+ distributed sensors.

Technical Details

  • Custom firmware written in C++ for ESP32 microcontrollers
  • MQTT broker for efficient message passing with QoS guarantees
  • Time-series database (PostgreSQL + TimescaleDB) for data storage
  • Real-time dashboard using WebSockets for live monitoring

Results

System has been running continuously for 18 months with 99.7% uptime. Processes over 1 million sensor readings daily with sub-second latency from sensor to dashboard.

Robotic Arm Controller

Python • OpenCV • Kinematics • ROS

Overview

Built a computer vision-guided robotic arm system capable of identifying, grasping, and sorting objects by color and shape. The system uses inverse kinematics for precise movement control.

Technical Details

  • 6-DOF arm with custom inverse kinematics solver
  • Object detection using OpenCV and custom trained YOLO model
  • Trajectory planning with collision avoidance
  • ROS integration for modular control architecture

Demo Video

Results

Achieved 92% accuracy in object sorting tasks with an average cycle time of 8 seconds per object. System can handle objects ranging from 2cm to 15cm in size.
