PRIMROSE - Lunar Excavation Rover

ANSYS • ONSHAPE • Team Leadership • Rapid Prototyping

My Role: Chassis Team Lead

Joined after Level 1 design, leading a 6-person subteam through fabrication, testing, and complete redesign of a 375 kg lunar excavation rover chassis. Responsible for all structural systems from concept through integration.

I joined after the PSTDL won $75,000 with their Phase 2 Level 1 proposal. The rover design was largely unfinalized and unmanufacturable at that point.

Level 2 vs Level 3 rover comparison showing evolution of design - from initial manufacturability fixes to complete chassis overhaul.

For Phase 2 Level 3 several failed aspects of the rover needed to be completely overhauled. Redesigned high load structures were verified with ANSYS as the team modified the rover for larger wheels and a wider wheelbase.

Design Philosophy: Speed Through Simplicity

Developed a manufacturing hierarchy to maximize build speed with limited resources:

  • 3D printed if possible (brackets, spacers)
  • Bent sheet metal + rivets (no welding required)
  • Bent sheet metal + welded joints
  • Simple miter cuts + welding
  • In-house machining only when unavoidable

This approach let us build fast and iterate faster - critical when discoveries during integration demanded immediate redesigns.

Level 2: Making It Work

Inherited a paper design that needed manufacturability fixes. Built the steel frame without time for FEA - pure experience-based design. Key contributions:

  • Resolved geometric interferences in CAD before cutting metal
  • Designed serviceable subassemblies (learned from automotive work)
  • Modified existing suspension hardware for 15-day endurance test
  • Result: 30,063m traversed, A-arm mounts being the only critical failure

Level 2 rover during endurance test showing original configuration

Level 3: Complete Chassis Overhaul

Chief engineer set new requirements: wider wheelbase for stability, larger wheels to prevent sinking in regolith, 30°/30° steering capability. My execution:

Structural Redesign:

  • Converted internal frame from steel to aluminum (weight savings for new electrical system)
  • Widened wheelbase through new suspension geometry
  • ANSYS FEA with aggressive FoS (verified against hand calcs)
  • Maintained top-down CAD architecture in Onshape for team coordination

Suspension Revolution:

  • Adapted existing kinematic layout to new wheel envelope
  • Converted failed bolted joints to shoulder bolt system
  • Precision-machined components where 3D printing couldn't meet tolerances
  • Accommodated steering actuators for crab-walk capability

New suspension assembly showing wider track width and improved mounting

Integration Under Fire

Fast design meant some components were modeled just days before fabrication. This required:

  • Constant communication with excavation/electrical teams
  • On-the-fly modifications when reality didn't match CAD
  • Balancing "perfect" vs "good enough" to meet deadlines

The payoff: 85% excavation rate improvement enabled by a chassis that could handle the increased loads.

Final Results & Reflection

The rover secured 3rd place overall at NASA Marshall, validating our scrappy approach against teams with significantly larger budgets. More importantly, this project transformed my engineering practice. The PSTDL gave me the freedom to fail fast, build faster, and learn from both. Working alongside a team that trusted a junior engineer to redesign critical systems taught me that good engineering isn't about perfection - it's about making the right compromises under pressure.

Team photo at NASA facility - every dent in that frame represents a lesson learned

Every dent in that frame represents a lesson learned. Every redesigned bracket reflects feedback from teammates who knew their systems better than I ever could. This wasn't just a competition rover - it was 18 months of compressed engineering education that shaped how I approach every project since.

IoT Sensor Network

C++ • MQTT • PostgreSQL • Docker

Overview

Designed and deployed a scalable IoT sensor network for environmental monitoring across multiple sites. The system collects temperature, humidity, air quality, and noise level data from 50+ distributed sensors.

Technical Details

  • Custom firmware written in C++ for ESP32 microcontrollers
  • MQTT broker for efficient message passing with QoS guarantees
  • Time-series database (PostgreSQL + TimescaleDB) for data storage
  • Real-time dashboard using WebSockets for live monitoring

Results

System has been running continuously for 18 months with 99.7% uptime. Processes over 1 million sensor readings daily with sub-second latency from sensor to dashboard.

Robotic Arm Controller

Python • OpenCV • Kinematics • ROS

Overview

Built a computer vision-guided robotic arm system capable of identifying, grasping, and sorting objects by color and shape. The system uses inverse kinematics for precise movement control.

Technical Details

  • 6-DOF arm with custom inverse kinematics solver
  • Object detection using OpenCV and custom trained YOLO model
  • Trajectory planning with collision avoidance
  • ROS integration for modular control architecture

Demo Video

Results

Achieved 92% accuracy in object sorting tasks with an average cycle time of 8 seconds per object. System can handle objects ranging from 2cm to 15cm in size.
